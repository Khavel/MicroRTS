# Contributing

## Minimal Diff Rule
Make the smallest possible change to satisfy the request.
Follow existing patterns exactly.
No refactors, renames, or structural changes unless explicitly requested.

## PR Rules
- One feature per PR.
- PR description must include:
  - Summary
  - How to test (exact steps)
  - Acceptance criteria checklist

## Code Rules
- Keep Unity diffs clean: text serialization, visible meta files.
- No committing Library/Temp/Build/etc.
